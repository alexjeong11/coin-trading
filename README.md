# 빗썸 자동매매 봇 실행 가이드 (Mac Mini 기준)

본 가이드는 작성된 `bot.py`를 맥미니(Mac Mini) 환경에서 24시간 백그라운드로 안전하게 구동하기 위한 방법을 안내합니다.

## 1. 초기 설정 (필수)

### 가상환경 생성 및 패키지 설치
맥미니의 터미널(Terminal)을 열고, 프로젝트 디렉토리로 이동한 뒤 아래 명령어를 차례대로 입력하세요.

```bash
# 1. 프로젝트 폴더로 이동 (경로는 본인 환경에 맞게 수정)
cd /Users/jeongcheol/Documents/ai-projects/coin-trading

# 2. 파이썬 가상환경(venv) 생성
python3 -m venv venv

# 3. 가상환경 활성화
source venv/bin/activate

# 4. 필수 패키지 설치
pip install -r requirements.txt
```

### API 키 설정 (.env 파일)
프로젝트 폴더 내에 있는 `.env.example` 파일을 복사하여 `.env` 파일을 생성합니다.

```bash
cp .env.example .env
nano .env
```
발급받은 빗썸 API의 `Connect Key`와 `Secret Key`를 입력하고 저장합니다. (Ctrl+O, Enter, Ctrl+X)

## 2. 봇 주요 기능 및 로직

*   **ATR 기반 다이내믹 그리드 (Dynamic Grid Spacing with ATR)**
    *   `python-bithumb` API 로 14일치 일봉 데이터를 조회하여 ATR(Average True Range) 변동성을 실시간 계산합니다.
    *   시장 변동성이 커지면 호가 간격을 최대 **0.5%**까지 제한하여 방어하고(기존 3%에서 축소), 횡보장에서는 최소 **0.25%**까지 초정밀하게 좁혀 회전율을 극대화합니다. 이는 변동성이 큰 장세에서도 '그물망'이 너무 넓어져 매매가 끊기는 현상을 방지합니다.
    *   **⚠️[중요] 수수료 쿠폰 필수**: 0.25% 간격 설정 시 빗썸 기본 수수료(0.25%)를 내면 손실이 납니다. 반드시 **빗썸 수수료 쿠폰(0.04% 적용)**을 구매/적용하셔야 왕복 0.08%를 제외한 0.17%의 순수익 창출이 가능합니다!
*   **초고속 스캘핑 5초 모니터링 루프**
    *   체결 후 10초 만에 꼬리(Wick)를 달고 반등하는 코인 시장의 특수상황(기회)을 놓치지 않기 위해, 봇은 **5초 주기**로 주문 체결 여부를 감시합니다. `0.25%`의 촘촘한 그물 코에서 즉각적으로 반대 포지션을 개설하기 위한 최적의 속도입니다.
*   **이익금 자동 재투자 (Auto-Compounding & Max Budget Ceiling)**
    *   매번 그리드를 구축할 때마다 빗썸 전체 평가 자산(코인+원화)을 측정하여 시드를 동적으로 늘려### 5. 이익금 재투자 상한선 (Max Budget Ceiling)
- 복리 모델에 의해 투자 예산이 점진적으로 증가하지만, 외부의 월급이나 대규모 추가 투자금을 보호하기 위한 상한선이 있습니다.
- `MAX_BUDGET` 설정값 (기본 20만원 등)을 초과하는 총 평가자산에 대해서는 봇이 자금을 끌어다 쓰지 않습니다.

### 6. 비용 안전 동적 리밸런싱 (Time-based Grid Reset / Trailing Grid)
- **왜 필요한가?**: 14일치 ATR(변동성)이 비정상적으로 높았던 직후 장세가 급격히 횡보(고착화)하면 그리드 간격이 너무 넓어져 매매가 전혀 일어나지 않는 좀비 그리드(Zombie Grid) 현상이 발생합니다.
- **방어 기제**: `bot.py`는 `RESET_TIMER_HOURS` (기본값: 6시간) 동안 단 한 건의 체결도 일어나지 않았을 경우 봇 본체가 장세를 재해석합니다.
- **비용 0원 달성**: 모든 '미체결 주문'만 강제 취소합니다. 매수하여 보유 중인 코인은 절대 손절가에 팔지 않고 홀딩(Holding)하며, 현재 시장 가격을 새로운 중심축(Center Price)으로 삼아 위/아래의 매도/매수 지망을 새롭게 재배포합니다. 

---*   **주문 상태 이중 검증 및 우아한 정지 (Graceful Pause)**
    *   빗썸 서버 문제로 주문이 증발(`cancel`)하면 동일한 주문을 재발송하며, 자산 현황 밸런스가 미세하게 깨져 특정 슬롯에 돈이 부족해지면 에러를 뿜는 대신 해당 슬롯만 조용히 일시 정지(Pause) 합니다. (이후 다른 슬롯이 팔려 돈이 생기면 자동 복구)
*   **추세 이탈 시 자동 셧다운 방어 (Out of Bounds Protection)**
    *   현재 가격이 최초 세팅된 10개의 최상단 혹은 최하단 그리드선(+/- 5%)을 뚫고 지나가버리면 봇이 스스로 멈추고 걸어둔 주문을 **일괄 취소** 한 뒤 방어선(그물 라인)을 새로운 가격 기준으로 스마트하게 재구축합니다.
*   **텔레그램 OpenClaw 연동 (로컬 API 서버)**
    *   별도의 `api_server.py`가 FastAPI로 구동되어 현재의 매매 현황, 총 자산(Total Asset KRW), 실시간 수익률(%), 미체결 주문 개수 등을 `/status` 로 반환하며 텔레그램 메신저에서 자유롭게 조회/중단할 수 있습니다.

## 3. 백그라운드 무중단 실행 & 자동 배포 (`deploy.sh`)

작성된 코드는 `bot.py`(자동매매 핵심 로직)와 `api_server.py`(텔레그램 연동 서버), 그리고 명령어를 관리하는 `openclaw_skill/SKILL.md` 로 구성되어 있습니다. 변경점을 반영하거나 재시작할 때는 항상 자체 제작된 배포 스크립트를 사용하세요.

### 🚀 봇 실행 및 코드 갱신 (단일 명령어)
```bash
# 터미널에서 가상환경 프로젝트 폴더 진입 후 한 줄만 입력
./deploy.sh
```

`deploy.sh`는 다음 3가지 작업을 자동으로 수행합니다:
1. 기존 백그라운드 봇과 서버 프로세스 완전히 검색 후 종료(Kill)
2. `openclaw_skill/SKILL.md` 의 수정 사항을 OpenClaw 텔레그램 연동 폴더로 자동 복사(동기화)
3. 봇과 서버 두 개를 새롭게 백그라운드(`nohup`)로 시작 (터미널 창을 닫아도 24시간 도망가지 않습니다!)

### 📝 실행 상태 및 로그 확인
봇은 두 가지 방식으로 로그를 남깁니다.
1. `nohup.out` (print문 및 시스템 에러 출력)
2. `trade.log` 및 `api.log` (봇 알고리즘과 API 서버에서 기록하는 상태 로그)
    - **로그 자동 분리 (Log Rotation)**: 로그 용량의 비대화를 막기 위해 매일 자정(0시)마다 로그 파일을 하루 단위(예: `trade.log.2023-10-25`)로 잘라서 보관합니다. 잘린 **과거 로그 파일들은 모니터링을 위해 자동으로 삭제되지 않도록 세팅**되어 있으니, 가끔 용량 확보를 위해 수동으로 지워주시면 됩니다.

실시간으로 매매가 체결되는지 보려면 아래 명령어를 사용하세요.
```bash
# 실시간 매매 로그 확인 (종료하려면 Ctrl + C)
tail -f trade.log
```

```bash
pkill -f 'bot\.py'

```

## (참고) Crontab을 사용하는 방식의 경우
만약 내부 `while True` 루프를 지우고 순수하게 1회 매매를 수행하는 스크립트로 변경하여, 맥의 스케줄러를 활용하고 싶다면 `crontab`을 셋팅할 수 있습니다.
그러나 맥(macOS)에서는 시스템 절전 모드 등의 이유로 `cron`보다 `launchd`를 권장하며, 개인용 봇의 경우 위에서 제시한 `nohup` + `while True` 루프 방식이 가장 설정이 간편하고 직관적입니다. (디스플레이가 꺼지더라도 Mac이 잠자기 모드에 들어가지 않도록 시스템 설정에서 "디스플레이가 꺼졌을 때 Mac이 자동으로 잠자지 않게 방지" 옵션을 켜두는 것을 권장합니다.)
